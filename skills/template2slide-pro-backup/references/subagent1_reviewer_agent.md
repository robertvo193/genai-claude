# Subagent 1: Reviewer Agent

**Purpose**: Validate generated outputs and provide feedback on content completeness, accuracy, and quality.

## Overview

The Reviewer Agent performs comprehensive validation of all outputs generated by the Slide Agent. It compares the original template content against the generated outputs (HTML, PPTX, PDF) to ensure completeness, accuracy, and quality.

## Input

- **Original template**: The input markdown template file
- **Generated outputs**:
  - `[Project_Name]_architecture_diagram.md`
  - `[Project_Name]_slide_structure.json`
  - `[Project_Name]_proposal.html` (HTML files)
  - `[Project_Name]_proposal.pptx`
  - `[Project_Name]_proposal.pdf`

## Actions

### 1. Extract Content from Template

Parse the original template to extract:
- All sections and their content
- All module information (name, type, purpose, alert_logic, preconditions, detection_criteria)
- All metadata (project name, client, dates, camera counts, etc.)
- Deployment method and architecture requirements
- Timeline milestones
- System requirements details

### 2. Extract Content from Generated Outputs

Parse each generated output to extract:

**From architecture_diagram.md**:
- Mermaid diagram code
- Deployment method represented
- Components and flows shown

**From slide_structure.json**:
- Total slide count
- All slide types and their content
- Module information for each module slide
- Timeline data
- All text content

**From HTML/PPTX/PDF**:
- Visible text content (slide by slide)
- Images and diagrams
- Formatting and layout
- Slide order and numbering

### 3. Compare and Validate

Perform comprehensive validation across multiple dimensions:

#### A. Content Completeness

Check that:
- ✅ All sections from template appear in outputs
- ✅ No sections missing or skipped
- ✅ All modules from template have corresponding slides
- ✅ All bullet points and details preserved
- ✅ Cover page, requirements, scope, architecture, implementation plan all present

#### B. Field Extraction Accuracy

For each module, verify:
- ✅ Module name matches template
- ✅ Module type (Standard/Custom) correctly extracted
- ✅ Purpose description matches template
- ✅ Alert trigger logic matches template
- ✅ Preconditions correctly extracted
- ✅ Detection criteria (if custom) matches template
- ✅ Image URL preserved (if present)
- ✅ Video URL preserved (if present)
- ❌ NO default/placeholder values in output fields

#### C. Module Information Completeness

For each module slide, check:
- ✅ All required fields present (purpose, alert_logic, preconditions)
- ✅ Custom modules have detection_criteria and data_requirements
- ✅ No empty fields unless image_url/video_url (which can be empty)
- ✅ Field content matches template exactly

#### D. Architecture Diagram Validation

- ✅ Deployment method matches template
- ✅ All components shown (cameras, NVR if applicable, AI system, dashboard)
- ✅ Flow direction correct (Camera → AI → Dashboard)
- ✅ All AI modules listed with full names
- ✅ Minimal, professional layout (no internal details like DB, API Gateway)
- ✅ Diagram renders without errors

#### E. HTML Validation

- ✅ All HTML files generate without errors
- ✅ Proper slide structure (title, content, media areas)
- ✅ No text overflow (content fits within boundaries)
- ✅ Proper typography (sizing, spacing, alignment)
- ✅ Dark theme applied (background.png, white text, viAct blue accents)
- ✅ Media elements embedded correctly (images, videos, diagrams)
- ✅ Smart content aggregation applied (merged short slides)

#### F. PPTX Validation

**Purpose**: Validate PowerPoint presentation for correctness and quality.

**File Integrity**:
- ✅ PPTX file is valid and opens correctly
- ✅ No corruption errors when opening file
- ✅ File size is reasonable (not excessively large)

**Slide Structure**:
- ✅ Total slide count matches expected number
  - Content slides count should match slide_structure.json
  - Reference slides should be inserted correctly (typically 25 additional slides)
  - Final total = content slides + reference slides
- ✅ Slide numbering continuous and correct (1, 2, 3, ...)
- ✅ No duplicate or missing slides

**Content Validation**:
- ✅ Text content matches template source
- ✅ All sections from template represented in slides
- ✅ No text overflow (content fits within slide boundaries)
  - Minimum 0.5" bottom margin maintained
  - No text cut off at edges
  - Proper word wrapping applied
- ✅ No layout issues or text truncation
  - Text boxes properly sized
  - Content not overlapping other elements
  - Proper spacing between elements

**Media and Visuals**:
- ✅ Images and diagrams are visible
- ✅ Architecture diagram embedded correctly (Slide 4)
  - Mermaid diagram converted to PNG
  - Transparent background applied
  - Proper sizing (not distorted)
  - All components clearly visible
- ✅ Module videos/images embedded correctly (Slides 11+)
  - Media downloaded from Google Drive URLs
  - Video files play correctly (if videos)
  - Images display correctly (if images)
  - Media priority logic applied (video_url → image_url → blank)
  - Proper aspect ratio maintained

**Formatting and Styling**:
- ✅ Formatting consistent across slides
- ✅ Dark theme applied correctly
  - Background image (background.png) visible on all slides
  - White text for readability
  - viAct blue accent colors (#00AEEF or similar)
- ✅ Typography standards met
  - Title slides: Large, uppercase, accent color
  - Body text: Minimum 11pt, readable size
  - Proper line height (1.4-1.6)
  - Consistent font family (Arial, Helvetica, or system UI)
- ✅ Proper margins and spacing
  - Minimum 0.5" bottom margin on all slides
  - Generous margins (content doesn't touch edges)
  - Consistent spacing between sections

**Special Slide Types**:
- ✅ Title slide (Slide 1)
  - Centered title
  - Date field populated (if available in template)
  - Clean, minimal design
- ✅ Two-column slide (Scope of Work)
  - Balanced left/right columns
  - Clear section headers
  - Equal or appropriate column widths
- ✅ Timeline slide (Implementation Plan)
  - Anti-overlap logic applied
  - No text overlap on timeline axis
  - Staggered heights for labels (if needed)
  - Proper phase/event/date formatting
- ✅ Module description slides
  - All required fields present (purpose, alert_logic, preconditions)
  - Media embedded on right side
  - Text content on left side
  - Proper 50/50 or 60/40 split

**Reference Slides**:
- ✅ Reference slides are inserted correctly
  - Architecture template slide inserted (after content slides)
  - Available slides 2-10 inserted after title slide
  - Available slides 11-25 inserted at end
  - Proper slide order maintained

#### G. PDF Validation

- ✅ PDF file opens without errors
- ✅ All pages present (matches PPTX slide count)
- ✅ Content matches PPTX
- ✅ Images and diagrams visible
- ✅ No formatting issues or corruption
- ✅ Text is searchable and selectable

#### H. Design Compliance

- ✅ Professional viAct branding applied
- ✅ Consistent color scheme (dark background, white text, blue accents)
- ✅ Proper margins (0.5" minimum bottom margin)
- ✅ Readable font sizes (minimum 11pt body, larger titles)
- ✅ Proper spacing and alignment
- ✅ Clean, minimalist aesthetic

## Output

### Validation Report

Generate a comprehensive validation report with:

**Summary**:
- Total templates processed
- Total outputs validated
- Overall pass/fail status
- Critical issues found (if any)

**Detailed Results** (per template):
- Content completeness: ✅ PASS / ❌ FAIL + details
- Field extraction accuracy: ✅ PASS / ❌ FAIL + details
- Module information completeness: ✅ PASS / ❌ FAIL + details
- Architecture diagram validation: ✅ PASS / ❌ FAIL + details
- HTML validation: ✅ PASS / ❌ FAIL + details
- PPTX validation: ✅ PASS / ❌ FAIL + details
- PDF validation: ✅ PASS / ❌ FAIL + details
- Design compliance: ✅ PASS / ❌ FAIL + details

**Issues Found** (if any):
- List each issue with:
  - Severity (Critical/High/Medium/Low)
  - Category (e.g., "Module Information Missing", "Architecture Incorrect")
  - Description
  - Location (template section, slide number, field name)
  - Expected vs Actual
  - Recommendation for fix

**Final Verdict**:
- ✅ **APPROVED**: All validations passed, ready for delivery
- ⚠️ **CONDITIONAL**: Minor issues, user approval required
- ❌ **REJECTED**: Critical issues, require regeneration

## Validation Rules

### Critical Fail Criteria (Reject Immediately)

1. **Missing Content**: Any template section not present in outputs
2. **Incorrect Module Information**: Any module field contains placeholder/wrong value
3. **Corrupted Files**: PPTX or PDF cannot be opened
4. **Missing Slides**: Slide count mismatch between JSON and PPTX/PDF
5. **Architecture Mismatch**: Deployment method or diagram incorrect

### Warning Criteria (Conditional Approval)

1. **Minor Formatting Issues**: Slight spacing/alignment problems
2. **Media Missing**: Image/video could not be downloaded (area left blank)
3. **Smart Aggregation**: Slides could be better merged (but functional)
4. **Text Sizing**: Some text slightly too small/large (but readable)

### Pass Criteria (Approved)

1. All content present and accurate
2. All fields correctly extracted
3. All files open and display correctly
4. Design compliant with standards
5. No critical or high-severity issues

## Workflow

```
For each set of template + outputs:
  1. Extract content from template
  2. Extract content from each output file
  3. Compare content completeness
  4. Validate field extraction accuracy
  5. Check module information completeness
  6. Validate architecture diagram
  7. Validate HTML rendering
  8. Validate PPTX generation
  9. Validate PDF conversion
  10. Check design compliance
  11. Generate validation report
  12. Return verdict (APPROVED/CONDITIONAL/REJECTED)
```

## Integration

This agent is invoked by the main orchestration layer (template2slide-pro SKILL.md) after the Slide Agent completes generation. The validation report is returned to the user for final approval.

## Dependencies

- **File reading**: Access to original template and all generated files
- **JSON parsing**: For slide_structure.json validation
- **PPTX reading**: For PowerPoint validation (python-pptx or similar)
- **PDF reading**: For PDF validation (PyPDF2 or similar)
- **HTML parsing**: For HTML validation (BeautifulSoup or similar)

## Troubleshooting

### Validation Fails - No Issues Found
- **Issue**: Validation fails but outputs appear correct
- **Solution**: Check validation logic thresholds, adjust criteria if too strict

### Cannot Extract Content from PPTX/PDF
- **Issue**: File reading library cannot parse file
- **Solution**: Verify file format, try alternative parsing library

### Module Fields Show as Empty
- **Issue**: Validation reports empty module fields
- **Solution**: Check slide_structure.json extraction logic in Slide Agent

### Architecture Diagram Validation Fails
- **Issue**: Mermaid diagram cannot be parsed or rendered
- **Solution**: Verify Mermaid syntax, check rendering service availability
