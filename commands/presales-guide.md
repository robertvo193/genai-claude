# Presales Guide Command

Display comprehensive guide for generating proposals from Deal Transfer Excel files.

## Purpose

Quick reference guide for presales team on how to:
- Generate proposal drafts from Excel
- Review and confirm placeholders
- Generate final PowerPoint and PDF deliverables

## When to Use

```
/presales-guide
```

Display this guide anytime you need help with the proposal workflow.

# Quick Start: Two Simple Commands

## Step 1: Generate Proposal Draft

```bash
/dealtransfer DT_0109.xlsx
```

**What happens:**
- System reads Deal Transfer Excel file
- Extracts customer information and requirements
- Generates proposal draft with 3 files
- Creates checklist of items to confirm

# What Each File Is

## Files Generated by /dealtransfer

### 1. `[Project]_template.md` **← Most Important**

**What it is:** Client-facing proposal document

**What you do with it:**
- Review the content for accuracy
- Fill in placeholder values (see checklist)
- Edit wording based on client knowledge
- Make any adjustments needed
- Save when ready for slides

**Example placeholder:**
```markdown
**Total Bandwidth:** 30 Mbps [NETWORK_001]
```

**After confirmation:**
```markdown
**Total Bandwidth:** 30 Mbps
```

---

### 2. `[Project]_reasoning.md`

**What it is:** Internal documentation of all decisions

**What you do with it:**
- Read to understand why certain decisions were made
- Reference if client asks questions
- Keep for internal records
- **Do NOT send to client**

---

### 3. `[Project]_checklist.md` **← Second Most Important**

**What it is:** List of all placeholders that need confirmation

**What you do with it:**
- Go through each item in the checklist
- Find the actual value for each placeholder
- Update the template file with confirmed values
- Check off items as you complete them

**Example checklist item:**
```
| ID | Section | Item | Content Estimated | Your Answer |
|----|---------|------|-------------------|-------------|
| NETWORK_001 | 5 | Internet bandwidth | 30 Mbps | |
```

**What you fill in:**
```
| NETWORK_001 | 5 | Internet bandwidth | 30 Mbps | 30 Mbps |
```

# The Manual Review Step (Critical!)

## Why Manual Review is Needed

The system generates a **draft** proposal, but some information needs your input:

**Common placeholders:**
- 📊 **Pricing** - Software licenses, implementation fees, maintenance
- 💻 **Hardware specs** - Exact CPU, RAM, GPU models
- 🌐 **Network** - Actual bandwidth available
- 👤 **Contacts** - Technical and commercial contact people
- ⚙️ **Configuration** - Alert escalation rules, retention periods

## How to Review

### 1. Open the Checklist

```bash
# Navigate to output directory
cd ./output/Leda_Inio_20260127_200428/

# Open checklist
code Leda_Inio_checklist.md
# or
nano Leda_Inio_checklist.md
```

### 2. Go Through Each Placeholder

For each item in the checklist:

**Example:**
```
ID: PRICING_SOFTWARE_001
Section: 9 - Commercial Terms
Item: Software license pricing
Estimated: [Not provided]
```

**What you do:**
1. Check pricing spreadsheet or talk to sales
2. Find actual software license price (e.g., $50,000)
3. Open template file
4. Find placeholder: `[Value] [PRICING_SOFTWARE_001]`
5. Replace with: `$50,000`

**Before:**
```markdown
**Software License:** [Value] [PRICING_SOFTWARE_001]
```

**After:**
```markdown
**Software License:** $50,000
```

### 3. Repeat for All Placeholders

Go through the entire checklist and fill in every placeholder.

### 4. Save Template

Save the template file when all placeholders are confirmed.

# Error Messages and What to Do

## Error: "File not found"

```
❌ File not found: DT_0109.xlsx
```

**What to do:**
1. Check file path is correct
2. Use full path if file is in different directory:
   ```bash
   /dealtransfer /home/user/documents/DT_0109.xlsx
   ```
3. Check file extension is `.xlsx` (not `.xls`)

---

## Error: "Template has placeholders"

```
❌ Template has placeholders that must be confirmed first

Found 5 placeholders:
  - NETWORK_001 (Section 5)
  - SERVER_CPU_001 (Section 5)
  - PRICING_SOFTWARE_001 (Section 9)
  - TERMS_PAYMENT_001 (Section 9)
  - CONTACT_TECHNICAL_001 (Section 11)
```

**What to do:**
1. Open the checklist file mentioned in error
2. Find actual values for each placeholder
3. Update template file (replace `[Value] [ID]` with actual value)
4. Save template
5. Run `/quotation` again

---

## Error: "Invalid Excel file"

```
❌ Invalid Deal Transfer Excel file: DT_0109.xlsx

Required sheets:
  - Commercial (S1)
  - Technical (S2)
```

**What to do:**
1. Open Excel file
2. Check for two sheets named "Commercial" and "Technical"
3. Sheet names must match exactly (case-sensitive: "Commercial" not "commercial")
4. Contact admin if file format is correct but system rejects it

# Tips and Best Practices

## Tip 1: Organize Your Output Directories

Each `/dealtransfer` run creates a new timestamped directory:
```
./output/
├── Leda_Inio_20260127_200428/
├── Conestoga_20260128_143052/
└── ABC_Corp_20260129_091530/
```

**Benefits:**
- Never overwrites previous work
- Easy to find latest version
- Can compare different deals

---

## Tip 2: Keep Original Excel Files Safe

**Best practice:**
- Save original Deal Transfer Excel in a safe location
- Don't edit the Excel file
- System only reads from Excel, never modifies it

---

## Tip 3: Use Reasoning File for Reference

The `reasoning.md` file explains **why** certain decisions were made:

**Example:**
```
Why 7 AI modules?
- Because pain points indicated need for PPE detection
- Use cases explicitly requested fire and vehicle detection
- Client concern about worker safety in danger zones

Why on-premise deployment?
- S2 Technical sheet specifies "On-premise"
- GDPR requirements mentioned in S2
- Client has stable internet but prefers local control
```

**Use this when:**
- Client asks questions about proposal
- You need to explain technical decisions
- Team reviews the proposal internally

---

## Tip 4: Fill Placeholders Completely

**WRONG:** Remove placeholder but leave incomplete
```markdown
**Software License:** TBD
```

**RIGHT:** Fill with actual value
```markdown
**Software License:** $50,000
```

**Why:** Placeholder detection looks for `[ID]` pattern, but incomplete information is still a problem!

---

## Tip 5: Save Multiple Versions if Needed

If you need to create variations:

```bash
# Generate version 1
/dealtransfer DT_0109.xlsx
# (edit and confirm as "Standard Package")
/quotation ./output/Leda_Inio_.../Leda_Inio_template.md

# Copy directory before editing
cp -r ./output/Leda_Inio_20260127_200428 ./output/Leda_Inio_Standard_20260127_200428

# Go back to original, edit for "Premium Package"
# (change pricing, add more modules)
/quotation ./output/Leda_Inio_20260127_200428/Leda_Inio_template.md

# Now you have two versions in separate directories
```

# What is Automatic vs Manual?

## Automatic (System Does It For You)

✅ **Extracts data from Excel** - Reads Commercial and Technical sheets
✅ **Determines AI modules** - Analyzes use cases and pain points
✅ **Generates proposal structure** - Creates all sections
✅ **Calculates estimates** - Bandwidth, hardware, timeline
✅ **Creates checklist** - Lists all items needing confirmation
✅ **Validates output** - Checks file format and completeness
✅ **Creates output directory** - Timestamped, organized
✅ **Generates slides** - Converts template to PPTX and PDF

## Manual (You Do It)

📋 **Review proposal content** - Check accuracy and completeness
📋 **Confirm placeholders** - Fill in pricing, contacts, specifications
📋 **Edit wording** - Adjust based on client knowledge
📋 **Approve final version** - Decide when ready for slides
📋 **Send to client** - Control final delivery

# FAQ

### Q: Can I skip the manual review step?

**A:** No, not recommended. The system generates a draft with placeholders that need your input. Running `/quotation` with placeholders will fail and tell you what to fix.

---

### Q: How long does the manual review take?

**A:** Typically 30-60 minutes:
- 15 minutes: Review template content
- 30 minutes: Confirm placeholders (pricing, contacts, specs)
- 15 minutes: Final edits and adjustments

---

### Q: Can I edit the template after generating slides?

**A:** Yes! Edit the template file and run `/quotation` again. It will regenerate the PPTX and PDF.

---

### Q: What if I make a mistake in the template?

**A:** Just edit the template and run `/quotation` again. You can regenerate slides as many times as needed.

---

### Q: Do I need to keep the reasoning file?

**A:** It's useful for reference but not required. Keep it for:
- Understanding why decisions were made
- Answering client questions
- Training new presales staff

---

### Q: Can I generate proposals without the Deal Transfer Excel?

**A:** No, the system requires the Deal Transfer Excel format with Commercial (S1) and Technical (S2) sheets. This ensures consistent data extraction.

---

### Q: What if the client has changes after I send the proposal?

**A:**
1. Edit the template file with changes
2. Run `/quotation` again to regenerate slides
3. Send updated version to client

You can update as many times as needed.
