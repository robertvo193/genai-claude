# Quotation Command

Generate final PowerPoint presentation and PDF document from verified proposal template.

## Purpose

Convert a verified proposal template (with all placeholders confirmed) into:
- **PowerPoint file (.pptx)** - For presentation and editing
- **PDF document (.pdf)** - For sharing and printing

## Pre-conditions

**Template must be verified:**
- ✅ All placeholders confirmed and replaced with actual values
- ✅ File exists and is readable markdown format
- ✅ Generated from /dealtransfer command (follows correct structure)

**What happens if placeholders remain:**
- ❌ Command fails with clear error
- 📋 Lists all placeholders found
- 🔧 Guides user to fix before proceeding

## Execution Flow

### Phase 1: Validate Input

1. Check argument provided (template file path)
2. Verify file exists and is .md format
3. Read file content for placeholder scanning

**Error handling:**
- If no argument: Show usage example
- If file not found: Display error with example
- If not markdown: Display file type error

### Phase 2: Scan for Placeholders

**Pattern to detect:** `[Value] [PLACEHOLDER_ID]`

**Examples of placeholders:**
- `30 Mbps [NETWORK_001]`
- `[Value] [PRICING_SOFTWARE_001]`
- `[Not provided] [CONTACT_TECHNICAL_001]`

**Scan logic:**
```bash
# Find all placeholder IDs
grep -o '\[[A-Z_]*[0-9]\]' template.md | sort -u
```

**If placeholders found:**
- Stop execution immediately
- Display error with all placeholders listed
- Show which section each placeholder is in
- Guide user to checklist and template files
- Exit with error code

**If no placeholders found:**
- Proceed to quotation_skill execution

### Phase 3: Extract Output Directory

1. Get directory of template file
2. Output PPTX and PDF to same directory
3. Extract project name from template content

**Example:**
```
Template: ./output/Leda_Inio_20260127_200428/Leda_Inio_template.md
Output:   ./output/Leda_Inio_20260127_200428/Leda_Inio.pptx
          ./output/Leda_Inio_20260127_200428/Leda_Inio.pdf
```

### Phase 4: Call quotation_skill

**NO confirmation required** - autonomous execution.

Use Skill tool to invoke quotation_skill:

```
Skill({
  skill: "quotation",
  args: "./output/Leda_Inio_20260127_200428/Leda_Inio_template.md"
})
```

**What the skill does:**
1. Parses template markdown structure
2. Converts markdown to PowerPoint slides
3. Generates PDF from PowerPoint
4. Creates two files in same directory:
   - `[ProjectName].pptx`
   - `[ProjectName].pdf`

### Phase 5: Validate and Summarize

1. Check both PPTX and PDF files created
2. Get file sizes
3. Display success message with locations
4. Indicate ready for client delivery

## Implementation Notes

### Placeholder Detection Algorithm

**Step 1: Read template file**
```bash
template_content=$(cat "$template_file")
```

**Step 2: Find all placeholder patterns**
```bash
# Pattern: [Value] [PLACEHOLDER_ID]
# Placeholder ID format: [A-Z_]*[0-9]
placeholders=$(echo "$template_content" | grep -oE '\[([A-Z_]+[0-9])\]' | sort -u)
```

**Step 3: Count and categorize**
```bash
placeholder_count=$(echo "$placeholders" | wc -l)

# Group by section (search for line numbers)
for ph in $placeholders; do
  line_number=$(grep -n "$ph" "$template_file" | cut -d: -f1)
  section=$(grep -B20 "$ph" "$template_file" | grep '^##' | tail -1)
  echo "- $ph ($section)"
done
```

### File Size Calculation

```bash
# Get human-readable sizes
pptx_size=$(ls -lh "$project_name.pptx" | awk '{print $5}')
pdf_size=$(ls -lh "$project_name.pdf" | awk '{print $5}')
```

### Project Name Extraction

**From template content:**
```bash
# Extract from "Project Owner:" field
project_name=$(grep "Project Owner:" "$template_file" | sed 's/**Project Owner:** //')
```

**Fallback to filename:**
```bash
# If not found in content, use filename
template_basename=$(basename "$template_file" _template.md)
project_name="$template_basename"
```

## User Experience

### Target Audience

Non-technical presales users who:
- Have reviewed and confirmed proposal template
- Filled in all placeholder values
- Ready to generate final client deliverables
- Want simple one-command slide generation

### Key Design Principles

1. **Safety first** - Prevents generating slides with incomplete info
2. **Clear errors** - Lists exactly what needs to be fixed
3. **One command** - No parameters to remember
4. **Same directory** - Outputs files alongside template
5. **No re-input** - Doesn't ask for information already in template

### Workflow Integration

This is **Step 2** of the two-step presales workflow:

```
Step 1: /dealtransfer <excel>  → Draft proposal
                ↓
         (Presales reviews and adjusts)
                ↓
Step 2: /quotation <template> → Final slides ← YOU ARE HERE
```

### Critical Boundary

**This command NEVER:**
- ❌ Reads Excel file
- ❌ Calls dealtransfer2template
- ❌ Modifies template content
- ❌ Creates new placeholders
- ❌ Regenerates from scratch

**This command ONLY:**
- ✅ Validates placeholders are gone
- ✅ Calls quotation_skill
- ✅ Generates PPTX and PDF
- ✅ Reports success

## Technical Constraints

### Input Requirements

**Must have:**
- Markdown format (.md)
- Generated by /dealtransfer (correct structure)
- No placeholder patterns present
- Required sections filled

**Cannot accept:**
- Other markdown files (user guide, readme, etc.)
- Templates from other sources
- Files with placeholders remaining
- Non-markdown files

### Output Files

**Generated by quotation_skill:**
- `[ProjectName].pptx` - PowerPoint format
- `[ProjectName].pdf` - PDF format

**Location:** Same directory as template file

**Overwrite behavior:**
- If PPTX/PDF exist, they are overwritten
- No backup created (user manages versions via output directories)

## Troubleshooting

### Problem: "Placeholders keep appearing"

**Cause:** Template still has `[Value] [PLACEHOLDER_ID]` patterns

**Solution:**
1. Open the checklist file
2. For each placeholder, find actual value
3. Search template for placeholder ID
4. Replace entire `[Value] [ID]` with actual value only

**Before:**
```markdown
**Total Bandwidth:** 30 Mbps [NETWORK_001]
```

**After:**
```markdown
**Total Bandwidth:** 30 Mbps
```

### Problem: "Wrong project name in output"

**Cause:** Template has incorrect "Project Owner" field

**Solution:**
1. Edit template "Project Owner" field
2. Re-run /quotation
3. Or manually rename output files

### Problem: "PPTX/PDF not created"

**Cause:** quotation_skill failed

**Solution:**
1. Check template format is correct
2. Ensure template was generated by /dealtransfer
3. Try regenerating from Excel with /dealtransfer
4. Contact admin if issue persists
